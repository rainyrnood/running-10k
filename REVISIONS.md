# 개정 이력 (Revision History)

10K 훈련 로그 앱(`index.html`)의 변경 내역을 기록합니다.

## 2026-05-31

### 1. 풋살 요일 선택 시 화면이 5/31로 튀던 버그 수정
- 증상: 6/1(W1) 주차에서 풋살 요일(수/목)을 고르면 화면이 자꾸 5/31(W0)로 되돌아감.
- 원인: 요일 선택 시 전체를 다시 그리는데, 이때 "오늘이 속한 주"만 펼치고 나머지를 모두 접도록 되어 있어, 방금 펼쳐 둔 주가 접히며 화면이 위로 올라감.
- 수정: 다시 그리기 전에 펼쳐 둔 주의 열림 상태를 저장해 두었다가 그대로 복원하도록 변경. 첫 렌더에서만 오늘 주차를 자동으로 펼침.

### 2. 5/31 같은 완전 휴식일에 "쉼 확인" 토글 추가
- 기존: 휴식일은 진행률 집계 대상이 아니라 체크표시 자체가 없었음.
- 변경: 휴식일에 "잘 쉬었음" 토글을 추가하고, 카드 우측에 체크표시(○/✔)를 표시.
- 주의: 휴식 확인은 전체 진행률(완료 N/총 세션) 계산에는 포함하지 않음. 목표 달성률이 왜곡되지 않도록 유지.

### 3. 입력 화면 간소화
- 기존: 카드를 펼치면 체중·워밍업/본운동/쿨다운·거리·페이스·컨디션·RPE·메모가 한꺼번에 노출되어 복잡했음.
- 변경: 기본 노출은 "완료 체크 + 메모"만. 거리·페이스·컨디션·RPE·체중은 "상세 입력" 접기 안으로 이동.
  - 이미 입력된 값이 있는 날은 상세 입력이 자동으로 펼쳐진 상태로 표시.
- 데이터·문구·계획 내용은 변경 없음. 보이는 방식만 정리.

### 4. "진행" 탭 신설
- 상단에 [로그 | 진행] 탭 추가.
- 진행 탭: 주차별로 집계 대상 세션(러닝·풋살·레이스)을 모아 날짜별로
  - 목표(계획 거리·페이스·목표 체중) vs 실제(기록한 거리·페이스·체중·RPE)
  - 달성 여부(○/✔)
  를 한눈에 보여줌. 상단에 전체 진행률 요약 표시.

### 5. 개정 이력 문서 신설
- 본 문서(`REVISIONS.md`)를 새로 작성.

### 6. (Rev_2) 풋살 요일 선택 시 화면이 5/31로 튀던 문제 재수정
- Rev_1에서 펼침 상태는 보존했지만, 요일 변경 시 화면 전체를 다시 그리면서 **스크롤 위치가 맨 위(5/31)로 리셋**되는 것이 남은 원인이었음.
- 수정: 주차 섹션 생성을 `weekSection()` 함수로 분리하고, 요일 변경 시 전체 재렌더 대신 **해당 주 섹션만 제자리에서 교체(replaceWith)**. 스크롤이 그대로 유지됨.

### 7. (Rev_2) "차트" 탭 신설 — 페이스 / 체중 추이
- [로그 | 진행 | 차트] 탭 구성으로 확장.
- 로그에 입력된 raw 데이터(`state.days`)를 그대로 사용. 외부 라이브러리 없이 내장 SVG로 그려 오프라인 동작 유지.
  - 체중 추이: 목표 체중 곡선(81→76kg 계획) 위에 실제 입력 체중을 겹쳐 표시.
  - 페이스 추이: 러닝 세션의 실제 평균 페이스를 세션 종류별 색(Easy/Long/Tempo/Interval/Race)으로 점 표시, 목표 5:15/km 기준선과 함께. 위로 갈수록 빠른 페이스.

### 8. (Rev_3) 페이스 차트 — 세션별 목표 반영
- 문제: 모든 세션을 단일 5:15 기준선과 비교해, 세션마다 목표가 다른데도(Easy 6:20 / Tempo 5:30→5:05 / Interval 4:45→4:40 등) 잘못 비교됨.
- 변경:
  - 복합 세션(Tempo/Interval/Race sim)에 메인 세트 목표 페이스 `targetSec`를 추가하고, 목표 페이스 조회 헬퍼 `sessionTargetSec()` 신설.
  - 페이스 차트를 재구성: **실제 = 채운 점, 그날 목표 = 빈 마커**(세션 종류별 색). 같은 종류끼리 목표를 선으로 이어 목표가 점점 빨라지는 진행 흐름을 표시. 5:15는 레이스 기준선으로만 유지.
  - 복합 세션은 기록을 입력하기 전에도 목표 마커가 보여 계획 진행을 파악 가능.
- 입력 보완: Tempo·Interval 등 복합 세션의 페이스 입력 라벨을 "본 구간(메인 세트) 평균 페이스"로 바꾸고 안내문 추가. 계획 대비 차이도 메인 세트 목표 기준으로 비교(목표보다 빠르면 달성 색).

### 9. (Rev_4) 페이스 차트 종류별 분리 + 전환 토글
- 문제: Easy/Long/Tempo/Interval/Race가 한 차트에 섞여 보기 어려움.
- 변경: 페이스 차트 위에 종류 전환 토글(Easy·Long·Tempo·Interval·Race)을 추가. 선택한 종류만 표시.
  - 선택 종류의 목표(빈 마커, 점선)와 실제(채운 점, 실선)를 각각 잇고, Y축은 그 종류 범위에 맞게 자동 확대.
  - 레이스 5:15 기준선은 해당 종류의 페이스 범위 안에 들어올 때만 표시(예: Tempo·Interval·Race).

### 10. (Rev_5) 페이스 차트 공통 고정 Y축 + 높이 1.5배
- 문제: 종류 전환 시 Y축이 매번 자동 재조정돼, 종류 간 페이스 위치를 비교할 수 없었음.
- 변경: 모든 종류가 **동일한 고정 Y축(4:30~6:45, 15초 눈금)**을 공유. 5:15 기준선도 항상 같은 높이.
  - Easy/Long은 하단 밴드, Tempo는 중간, Interval은 상단 밴드에 고정되어 토글만으로 종류 간 차이를 한눈에 파악.
  - 넓은 범위가 눌리지 않도록 차트 높이를 기존(240) 대비 1.5배(340)로 키움.
  - 프리셋 범위를 벗어나는 값은 축 가장자리로 클램프.
